2. API ROUTES (add to server/routes.ts)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Route 1: GET /api/pm-pages/:pageId/comments
Returns all comments for a page, ordered by createdAt ascending. Include user display name for each comment. Filter: only return comments the user has permission to see (they are a page viewer/editor or the comment author).

Route 2: POST /api/pm-pages/:pageId/comments
Creates a comment. Request body: z.object({ content: z.string(), blockId: z.number().nullable().optional(), parentCommentId: z.number().nullable().optional(), mentions: z.array(z.string()).optional() }). For each userId in mentions array, create a notification of type "mention" for that user with message "[commenter name] mentioned you in [page title]" and linkUrl "/dashboard/pm-editor/[pageId]". Return created comment.

Route 3: PATCH /api/pm-comments/:id
Updates a comment's content or isResolved. Request body: z.object({ content: z.string().optional(), isResolved: z.boolean().optional() }). Only allow if user is the comment author (for content) or has edit access to the page (for resolving). Return updated comment.

Route 4: DELETE /api/pm-comments/:id
Deletes a comment and all its replies (where parentCommentId = this id). Only allow if user is the author. Return { success: true }.

Route 5: POST /api/pm-pages/:pageId/versions
Creates a version snapshot. Reads all blocks for the page and serializes them. Request body: z.object({ changeDescription: z.string().optional() }). Set versionNumber to max(versionNumber) + 1 for this pageId. Return created version.

Route 6: GET /api/pm-pages/:pageId/versions
Returns all versions for a page, ordered by versionNumber descending. Limit to 50. Each version returns id, versionNumber, title, changeDescription, createdAt (not the full blocksSnapshot — that's fetched on demand).

Route 7: GET /api/pm-page-versions/:id
Returns a single version with its full blocksSnapshot.

Route 8: POST /api/pm-page-versions/:id/restore
Restores a page to a version. Creates a new version with the current state first (auto-save with changeDescription "Auto-saved before restore"). Then replaces all blocks on the page: delete all current blocks, then create new blocks from the version's blocksSnapshot. Return the restored page.

Route 9: GET /api/pm-notifications
Returns notifications for the current user, ordered by createdAt descending. Limit to 50. Accept query param unreadOnly (boolean) — if true, only return where isRead is false.

Route 10: PATCH /api/pm-notifications/:id/read
Marks a notification as read. Set isRead to true. Return updated notification.

3. AUTO-VERSIONING

Update PATCH /api/pm-blocks/:id to auto-create a version every 30 edits. Keep a counter per page (use a simple in-memory Map<pageId, editCount> in server/routes.ts). When editCount reaches 30, call the version creation logic internally with changeDescription "Auto-saved", reset counter to 0.

Add a "Comments" button (MessageSquare icon) in the top-right of pm-editor.tsx (next to the Share button). Clicking it toggles a right sidebar panel (320px wide).

Comments panel layout:
- Header: "Comments" + close button (X icon) + toggle "Show resolved"
- Comment list: each comment shows user avatar/initials, user name, timestamp (relative), comment text, "Reply" button, "Resolve" button (if unresolved), three-dot menu (Edit, Delete — only for own comments)
- Replies are indented 24px under their parent comment, connected by a thin vertical line (border-l)
- At the bottom: a comment input area with a textarea, @mention autocomplete (type @ to show user list dropdown), and a "Comment" button
- Block-level comments: if a comment has a blockId, show a small link "On: [block preview]" below the comment. Clicking it scrolls to and highlights that block.

@mention autocomplete: When user types "@" in the comment textarea, show a dropdown of workspace members. Filter as user types after "@". Selecting a user inserts their name and adds their userId to the mentions array.

Data fetching:
- queryKey: ["pm-comments", pageId]
- endpoint: GET /api/pm-pages/:pageId/comments

Mutations:
- Create comment: POST /api/pm-pages/:pageId/comments, invalidate ["pm-comments", pageId]
- Update comment: PATCH /api/pm-comments/:id, invalidate ["pm-comments", pageId]
- Delete comment: DELETE /api/pm-comments/:id, invalidate ["pm-comments", pageId]

Data fetching:
- queryKey: ["pm-page-versions", pageId]

Update these existing routes to create notifications:

- POST /api/pm-pages/:pageId/permissions (when sharing with a user): Create notification type "share" for the target user: "[sharer name] shared [page title] with you"
- POST /api/pm-workspaces/:workspaceId/members (when inviting): Create notification type "invite" for the invited user (if they exist): "[inviter name] invited you to [workspace name]"
- POST /api/pm-pages/:pageId/comments: Already handled above (mention notifications). Also create a "comment" notification for the page owner (if commenter is not the owner): "[commenter name] commented on [page title]"
