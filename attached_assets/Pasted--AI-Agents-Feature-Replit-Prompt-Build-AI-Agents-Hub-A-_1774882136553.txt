# AI Agents Feature — Replit Prompt

Build **AI Agents Hub** — A full-featured AI agent workspace where users create projects, select an external AI provider (ChatGPT, Gemini, Perplexity, Claude), and chat with that provider through a professional multi-panel interface with chat history sidebar, project management, and new-chat creation.

## 2. API ROUTES

Add all routes inside `server/routes.ts`. Every route uses `isAuthenticated` middleware and filters by `eq(table.userId, req.user.claims.sub)`.

### 2.1 Project Routes

**GET `/api/ai-agent-projects`** — Returns all non-archived projects for the authenticated user, ordered by `createdAt` desc.

```ts
app.get("/api/ai-agent-projects", isAuthenticated, async (req, res) => {
  const projects = await db.select().from(aiAgentProjects)
    .where(and(eq(aiAgentProjects.userId, req.user.claims.sub), eq(aiAgentProjects.isArchived, false)))
    .orderBy(desc(aiAgentProjects.createdAt));
  res.json(projects);
});
```

**POST `/api/ai-agent-projects`** — Creates a new project. Validates body with `insertAiAgentProjectSchema`. Sets `userId` from session.

```ts
app.post("/api/ai-agent-projects", isAuthenticated, async (req, res) => {
  const parsed = insertAiAgentProjectSchema.parse({ ...req.body, userId: req.user.claims.sub });
  const [project] = await db.insert(aiAgentProjects).values(parsed).returning();
  res.json(project);
});
```

**PATCH `/api/ai-agent-projects/:id`** — Updates project name, description, provider, model, or systemPrompt. Validates that the project belongs to the user.

Request body Zod schema:
```ts
z.object({
  name: z.string().min(1).max(100).optional(),
  description: z.string().max(1000).optional(),
  provider: z.enum(["chatgpt", "gemini", "perplexity", "claude"]).optional(),
  model: z.string().min(1).max(100).optional(),
  systemPrompt: z.string().max(5000).optional(),
})
```

**DELETE `/api/ai-agent-projects/:id`** — Soft-deletes the project by setting `isArchived` to `true`. Also soft-deletes all chats under this project.

### 2.2 Chat Routes

**GET `/api/ai-agent-projects/:projectId/chats`** — Returns all non-archived chats for a specific project belonging to the user, ordered by `updatedAt` desc.

**POST `/api/ai-agent-projects/:projectId/chats`** — Creates a new chat under the given project. Sets `provider` and `model` from the project defaults. Validates body:
```ts
z.object({ title: z.string().min(1).max(200).optional() })
```

**PATCH `/api/ai-agent-chats/:id`** — Updates chat title. Validates owner. Body:
```ts
z.object({ title: z.string().min(1).max(200) })
```

**DELETE `/api/ai-agent-chats/:id`** — Soft-deletes the chat by setting `isArchived` to `true`.

### 2.3 Message Routes

**GET `/api/ai-agent-chats/:chatId/messages`** — Returns all messages for a chat belonging to the user, ordered by `createdAt` asc.

**POST `/api/ai-agent-chats/:chatId/messages`** — Sends a user message, calls the AI provider via `aiService`, saves both user and assistant messages, and returns the assistant message. Validates body:
```ts
z.object({ content: z.string().min(1).max(10000) })
```

Implementation details for this route:
1. Verify the chat belongs to the user.
2. Fetch the chat to get `provider` and `model`.
3. Fetch the parent project to get `systemPrompt`.
4. Save the user message to `aiAgentMessages` with `role: "user"`.
5. Fetch all previous messages for this chat (ordered by `createdAt` asc) to build the conversation context.
6. Call `aiService.generateText` with the system prompt and all messages as context.
7. Save the assistant response to `aiAgentMessages` with `role: "assistant"`.
8. Update the chat `updatedAt` to `new Date()`.
9. If this is the first message in the chat (only 1 user message exists before this), auto-generate a short title: call `aiService.generateText` with system prompt `"Generate a 3-6 word title for this conversation. Return only the title text, nothing else."` and user prompt being the user's first message. Update the chat title with the result.
10. Return the assistant message object as JSON.

**POST `/api/ai-agent-chats/:chatId/messages/stream`** — Same as above but uses `aiService.generateStream` and returns a Server-Sent Events stream. The frontend will use `EventSource` or `fetch` with `ReadableStream` to consume the response.

### 2.4 Provider Info Route

**GET `/api/ai-agent-providers`** — Returns a static JSON array of available providers:
```ts
[
  { id: "chatgpt", name: "ChatGPT", icon: "bot", models: ["gpt-4o-mini", "gpt-4o", "gpt-4-turbo"] },

]
```

This route does NOT require `isAuthenticated`.

### 3.8 Mutations

- **Create project**: `POST /api/ai-agent-projects`. On success, invalidate `["/api/ai-agent-projects"]`.
- **Update project**: `PATCH /api/ai-agent-projects/:id`. On success, invalidate `["/api/ai-agent-projects"]`.
- **Delete project**: `DELETE /api/ai-agent-projects/:id`. On success, invalidate `["/api/ai-agent-projects"]`. Reset selected project and chat to null.
- **Create chat**: `POST /api/ai-agent-projects/:projectId/chats`. On success, invalidate `["/api/ai-agent-projects", projectId, "chats"]`. Set the new chat as selected.
- **Update chat**: `PATCH /api/ai-agent-chats/:id`. On success, invalidate `["/api/ai-agent-projects", selectedProjectId, "chats"]`.
- **Delete chat**: `DELETE /api/ai-agent-chats/:id`. On success, invalidate `["/api/ai-agent-projects", selectedProjectId, "chats"]`. Reset selected chat to null.
- **Send message**: `POST /api/ai-agent-chats/:chatId/messages`. On success, invalidate `["/api/ai-agent-chats", chatId, "messages"]` and `["/api/ai-agent-projects", selectedProjectId, "chats"]` (because chat title may have changed).

## 5. AI FEATURES

### 5.1 Chat Completion

Use `aiService.generateText` for the main chat functionality.

**System prompt**: Use the project's `systemPrompt` field value (user-customizable per project).

**User prompt**: Build the conversation context by mapping all previous messages into the standard `{ role, content }` array format, then append the new user message.

**Model**: `gpt-4o-mini`

**Response format**: Plain text string (the assistant's reply).

### 5.2 Auto-Title Generation

Use `aiService.generateText` when the first message is sent in a new chat.

**System prompt**: `"Generate a concise 3-6 word title that summarizes the following conversation starter. Return only the title text with no quotes, no punctuation at the end, and no extra explanation."`

**User prompt**: The content of the user's first message.

**Model**: `gpt-4o-mini`

**Response format**: Plain text string (used to update `aiAgentChats.title`).

- `Enter` in the message textarea: sends the message (calls the send mutation).
- `Shift+Enter` in the message textarea: inserts a newline.
- `Escape` in any open dialog: closes the dialog.

### 7.4 Error Handling

- If `POST /api/ai-agent-chats/:chatId/messages` fails, show a toast notification using the existing toast system with the message: "Failed to send message. Please try again."
- If project creation fails, show toast: "Failed to create project. Please try again."
- If chat creation fails, show toast: "Failed to create chat. Please try again."
- All API routes return `{ error: string }` with status 400 for validation errors, 404 if the resource is not found or does not belong to the user, and 500 for server errors.

### 7.5 Responsive Behavior

- On screens below 768px width, the left sidebar collapses into a slide-over drawer triggered by a hamburger button in the top-left of the main content area.
- The message area uses `max-w-[80%]` on desktop and `max-w-[90%]` on mobile for message bubbles.
